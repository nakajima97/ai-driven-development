# 要件定義書: 管理者向けユーザー管理画面

## 1. 概要

管理者がシステム上のユーザーを効率的に管理するための管理画面を実装する。
一覧表示・検索・停止の3機能を提供する。

**技術スタック**: Next.js (App Router) + TypeScript + Tailwind CSS

---

## 2. ユーザーストーリー

| # | ストーリー |
|---|-----------|
| US-01 | 管理者として、登録済みユーザーの一覧を確認したい。全体の状況を把握するため。 |
| US-02 | 管理者として、名前やメールアドレスでユーザーを検索したい。特定のユーザーを素早く見つけるため。 |
| US-03 | 管理者として、問題のあるユーザーのアカウントを停止したい。不正利用を防ぐため。 |
| US-04 | 管理者として、停止済みのユーザーを再有効化したい。誤停止を修正するため。 |

## 3. 画面仕様

### 3.1 ページ: ユーザー管理画面 (`/admin/users`)

#### レイアウト

```
┌─────────────────────────────────────────────────────┐
│  管理画面                                  [管理者名] │
├─────────────────────────────────────────────────────┤
│  ユーザー管理                                         │
│                                                     │
│  [🔍 名前・メールアドレスで検索...]  [検索]           │
│  ステータス: [すべて ▼]                               │
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │ ID │ 名前 │ メール │ 登録日 │ ステータス │ 操作 │  │
│  ├───────────────────────────────────────────────┤  │
│  │  1 │ 田中 │ ...   │ ...   │ 有効       │[停止]│  │
│  │  2 │ 鈴木 │ ...   │ ...   │ 停止中     │[有効]│  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  [前へ]  1 / 10  [次へ]                              │
└─────────────────────────────────────────────────────┘
```

#### 検索フィールド

| 項目 | 仕様 |
|------|------|
| 入力フォーム | テキスト入力（プレースホルダー: "名前・メールアドレスで検索..."） |
| 検索トリガー | 検索ボタンクリック または Enterキー |
| 最小文字数 | 1文字以上 |
| リアルタイム検索 | 任意（300msデバウンス推奨） |

#### ステータスフィルター

| 値 | 表示ラベル |
|----|-----------|
| `all` | すべて（デフォルト） |
| `active` | 有効のみ |
| `suspended` | 停止中のみ |

#### ユーザーテーブル

| カラム | 内容 |
|--------|------|
| ID | ユーザーID（数値） |
| 名前 | 表示名 |
| メールアドレス | ログイン用メール |
| 登録日 | `YYYY/MM/DD` 形式 |
| ステータス | `有効` / `停止中`（バッジ表示） |
| 操作 | 有効 → [停止] ボタン / 停止中 → [有効化] ボタン |

#### ページネーション

| 項目 | 仕様 |
|------|------|
| 1ページあたりの件数 | 20件 |
| 表示情報 | 現在ページ / 総ページ数 |
| 操作 | 前へ / 次へ ボタン |

## 4. 機能要件

### 4.1 ユーザー一覧表示

- 全ユーザーを登録日の降順で表示する
- ページネーションにより20件ずつ表示する
- テーブルはローディング中にスケルトン表示を行う
- 0件の場合は「ユーザーが見つかりません」と表示する

### 4.2 検索

- 名前・メールアドレスの部分一致で検索する（大文字小文字を無視）
- 検索クエリはURLクエリパラメータ（`?q=`）に反映し、ブックマーク・共有できる
- ステータスフィルターと組み合わせて絞り込みできる
- 検索後にページネーションは1ページ目にリセットされる

### 4.3 ユーザー停止

- 停止操作を行う前に確認ダイアログを表示する
- 停止後、該当ユーザーのステータスを「停止中」に更新し、一覧をリフレッシュする
- 停止中のユーザーは対象のサービスにログインできない
- 停止したユーザーは「有効化」ボタンで再有効化できる
- 自分自身（ログイン中の管理者）は停止できない

---

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| 認証 | 管理者権限を持つユーザーのみアクセス可能（未認証はログイン画面にリダイレクト） |
| 権限チェック | サーバーサイド（Middleware or Server Component）でロール検証を行う |
| レスポンス速度 | 一覧の初回表示は1秒以内 |
| アクセシビリティ | ボタンに適切な `aria-label` を付与する |
| レスポンシブ | デスクトップ（1024px以上）を主対象。モバイルは横スクロール対応 |
| エラーハンドリング | API失敗時はトースト通知でエラーを表示する |

## 6. API仕様

### GET `/api/admin/users`

ユーザー一覧を取得する。

**クエリパラメータ**

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `q` | string | - | 検索キーワード（名前・メール） |
| `status` | `all` \| `active` \| `suspended` | - | ステータスフィルター（デフォルト: `all`） |
| `page` | number | - | ページ番号（デフォルト: 1） |
| `limit` | number | - | 1ページあたりの件数（デフォルト: 20） |

**レスポンス（200 OK）**

```json
{
  "users": [
    {
      "id": 1,
      "name": "田中 太郎",
      "email": "tanaka@example.com",
      "createdAt": "2024-01-15T00:00:00Z",
      "status": "active"
    }
  ],
  "pagination": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "totalPages": 5
  }
}
```

---

### PATCH `/api/admin/users/[id]/status`

ユーザーのステータスを変更する。

**リクエストボディ**

```json
{
  "status": "suspended"
}
```

| フィールド | 型 | 値 |
|-----------|-----|-----|
| `status` | string | `"active"` または `"suspended"` |

**レスポンス**

| ステータスコード | 説明 |
|----------------|------|
| 200 OK | 更新成功。更新後のユーザーオブジェクトを返す |
| 400 Bad Request | 不正なステータス値 |
| 403 Forbidden | 権限なし、または自分自身を停止しようとした |
| 404 Not Found | ユーザーが存在しない |

## 7. データモデル

```typescript
type UserStatus = "active" | "suspended";

interface AdminUser {
  id: number;
  name: string;
  email: string;
  createdAt: string; // ISO 8601
  status: UserStatus;
}

interface PaginationMeta {
  total: number;
  page: number;
  limit: number;
  totalPages: number;
}

interface GetUsersResponse {
  users: AdminUser[];
  pagination: PaginationMeta;
}
```

## 9. 受け入れ基準

| # | 条件 |
|---|------|
| AC-01 | 管理者でログインした状態で `/admin/users` にアクセスするとユーザー一覧が表示される |
| AC-02 | 未認証状態でアクセスするとログイン画面にリダイレクトされる |
| AC-03 | 管理者以外のロールでアクセスすると403エラー画面が表示される |
| AC-04 | 検索フォームにキーワードを入力して検索すると、名前・メールに一致するユーザーのみ表示される |
| AC-05 | ステータスフィルターで「停止中のみ」を選択すると停止中ユーザーのみ表示される |
| AC-06 | 有効なユーザーの[停止]ボタンをクリックすると確認ダイアログが表示される |
| AC-07 | 確認ダイアログで「停止する」をクリックするとステータスが「停止中」に変わる |
| AC-08 | 停止中ユーザーの[有効化]ボタンをクリックするとステータスが「有効」に戻る |
| AC-09 | ログイン中の管理者自身には停止ボタンが表示されない（または非活性） |
| AC-10 | ページネーションの「次へ」「前へ」で正しくページが切り替わる |

---

## 10. 対象外（スコープ外）

- ユーザーの詳細情報の編集（名前・メール変更など）
- ユーザーの削除（物理削除）
- ユーザーへの通知メール送信
- 管理者アカウントの管理
- 操作ログ・監査ログの記録・表示
- CSV エクスポート
