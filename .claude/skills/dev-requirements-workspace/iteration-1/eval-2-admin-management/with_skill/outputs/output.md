# 管理者ユーザー管理画面 画面要件定義書

## 目的

管理者がシステム上のユーザーを一覧表示・検索・停止できる管理画面を提供する。
ユーザーの状態把握と不正利用や問題のあるアカウントへの迅速な対処を可能にする。

## 対象ユーザー

- **システム管理者**: サービス全体のユーザーを管理する権限を持つ内部スタッフ

## 画面の要否判断

**画面が必要**: ユーザー一覧閲覧・検索・停止操作という複数のインタラクションが必要であり、視覚的なインターフェースが必須。

---

## 画面構成

### 1. ユーザー管理一覧画面（`/admin/users`）

- **役割**: ユーザーの一覧を表示し、検索・フィルタリング・個別操作を行う主画面
- **主要要素**:
  - ページタイトル「ユーザー管理」
  - 検索バー（メールアドレス・ユーザー名で検索）
  - フィルター（ステータス: 全て / アクティブ / 停止済み）
  - ユーザーテーブル（ユーザーID・ユーザー名・メールアドレス・登録日・ステータス・操作ボタン）
  - ページネーション（1ページあたり20件）
  - ローディングスケルトン（データ取得中）
  - 空状態メッセージ（検索結果なし）
- **遷移先**: ユーザー詳細画面（`/admin/users/[id]`）、停止確認モーダル

### 2. ユーザー詳細画面（`/admin/users/[id]`）

- **役割**: 特定ユーザーの詳細情報を確認し、停止・再有効化の操作を行う
- **主要要素**:
  - ユーザー基本情報（ユーザー名・メールアドレス・登録日・最終ログイン日時）
  - 現在のステータス表示（バッジ: アクティブ / 停止済み）
  - 停止 / 再有効化ボタン（現在のステータスに応じて切り替え）
  - 戻るリンク（一覧画面へ）
  - ローディング・エラー状態
- **遷移先**: ユーザー管理一覧画面（戻る操作）、停止確認モーダル

### 3. 停止確認モーダル（共通コンポーネント）

- **役割**: ユーザー停止・再有効化操作の誤操作防止のための確認ダイアログ
- **主要要素**:
  - 確認メッセージ（「[ユーザー名] を停止しますか？」）
  - キャンセルボタン
  - 実行ボタン（停止 / 再有効化）
  - 処理中スピナー（API呼び出し中）
  - エラーメッセージ（操作失敗時）

## 機能リスト

| 機能 | 優先度 | 備考 |
|------|--------|------|
| ユーザー一覧表示 | Must | ページネーション付き（20件/ページ） |
| ユーザー名・メールアドレスによる検索 | Must | 部分一致、デバウンス処理（300ms） |
| ステータスによるフィルタリング | Must | アクティブ / 停止済み / 全て |
| ユーザー詳細表示 | Must | 一覧→詳細への画面遷移 |
| ユーザー停止 | Must | 確認モーダルあり |
| ユーザー再有効化（停止解除） | Must | 確認モーダルあり |
| 停止操作の成功通知（トースト） | Should | 操作完了後にフィードバック表示 |
| ソート機能（登録日・ユーザー名） | Should | テーブルヘッダーのクリックでソート |
| 管理者権限チェック（未認証リダイレクト） | Must | `/login` へリダイレクト |
| エラー状態の表示 | Must | API失敗時にエラーメッセージ表示 |
| 空状態の表示 | Must | 検索結果0件時のメッセージ |
| ローディング状態の表示 | Must | スケルトンUI |
| 操作ログ表示（停止履歴） | Nice to Have | 誰がいつ停止したか確認できる |
| CSV エクスポート | Nice to Have | ユーザー一覧のダウンロード |

---

## バリデーション要件

- 検索バー: 入力長は最大100文字
- 停止操作: 自分自身（ログイン中の管理者）は停止不可
- 停止操作: すでに停止済みのユーザーへの停止操作はUI側で無効化

---

## 状態一覧

| 状態 | 対象 | 内容 |
|------|------|------|
| ローディング | 一覧・詳細 | スケルトンUIを表示 |
| 空状態 | 一覧 | 「ユーザーが見つかりません」メッセージ |
| エラー状態 | 一覧・詳細・モーダル | エラーメッセージとリトライ導線 |
| 操作処理中 | モーダル | ボタンをdisabledにしてスピナー表示 |
| 操作成功 | モーダル後 | トースト通知を表示してモーダルを閉じる |

---

## 非機能要件

- **パフォーマンス**: 一覧の初期表示は2秒以内。検索は入力から1秒以内に結果を反映（デバウンス後）
- **アクセシビリティ**: キーボード操作対応（Tabフォーカス、Enterキー操作）。スクリーンリーダーのためのaria-label付与。色だけに依存しないステータス表示（テキスト + バッジ色）
- **セキュリティ**: 管理者ロールを持つユーザーのみアクセス可能。未認証・権限なしの場合は適切なページへリダイレクト
- **レスポンシブ**: デスクトップ優先（1024px以上）。タブレット対応（768px以上）は Should

---

## 技術スタック前提

- Next.js（App Router）+ TypeScript + Tailwind CSS
- サーバーコンポーネント（一覧・詳細の初期データ取得）
- クライアントコンポーネント（検索・フィルター・モーダル操作）

## 確認事項（未決定・要確認）

以下の点はユーザーに確認が必要です:

1. **ユーザー停止の定義**: 停止済みユーザーはログインできなくなる想定でよいか？それともコンテンツ投稿のみ制限するか？
2. **権限管理の粒度**: 管理者ロールは単一か、複数レベル（スーパー管理者 / 一般管理者）があるか？
3. **停止通知**: ユーザーが停止された際にメール通知を送るか？
4. **ページネーションの方式**: 従来のページ番号方式か、無限スクロールか？
5. **詳細画面の必要性**: 一覧画面だけで十分か、詳細画面（`/admin/users/[id]`）も作るか？
