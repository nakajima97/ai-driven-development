# ユーザープロフィール編集画面 要件定義書

## 1. 概要

ユーザーが自分のプロフィール情報（名前・アイコン・自己紹介）を編集できる画面を提供する。

## 2. 技術スタック

- Next.js (App Router)
- TypeScript
- Tailwind CSS

## 3. ユーザーストーリー

- ユーザーとして、自分の表示名を変更したい
- ユーザーとして、プロフィールアイコン画像をアップロード・変更したい
- ユーザーとして、自己紹介文を編集したい
- ユーザーとして、変更内容をプレビューしてから保存したい
- ユーザーとして、変更に失敗した場合にエラーメッセージを確認したい

## 4. 画面仕様

### 4.1 ページ情報

| 項目 | 内容 |
|------|------|
| ルート | `/profile/edit` |
| コンポーネント | `app/profile/edit/page.tsx` |
| アクセス制御 | 認証済みユーザーのみ |

### 4.2 レイアウト

```
+-------------------------------+
|  ヘッダー（ページタイトル）       |
+-------------------------------+
|                               |
|  [アイコン画像]                 |
|  [画像変更ボタン]               |
|                               |
|  表示名                        |
|  [テキスト入力]                 |
|                               |
|  自己紹介                      |
|  [テキストエリア]               |
|                               |
|  [保存ボタン]  [キャンセル]      |
|                               |
+-------------------------------+
```

### 4.3 入力フィールド仕様

#### 表示名（name）

| 項目 | 仕様 |
|------|------|
| 入力タイプ | `text` |
| 必須 | はい |
| 最小文字数 | 1文字 |
| 最大文字数 | 50文字 |
| バリデーションメッセージ | 「名前を入力してください」「50文字以内で入力してください」 |

#### アイコン画像（avatar）

| 項目 | 仕様 |
|------|------|
| 入力タイプ | `file`（非表示 input + カスタムボタン） |
| 許可形式 | JPEG / PNG / WebP |
| 最大サイズ | 5MB |
| プレビュー | アップロード前にブラウザ上でプレビュー表示 |
| デフォルト | 現在のアイコン、または未設定の場合はデフォルトアバター |
| バリデーションメッセージ | 「対応していないファイル形式です」「ファイルサイズは5MB以内にしてください」 |

#### 自己紹介（bio）

| 項目 | 仕様 |
|------|------|
| 入力タイプ | `textarea` |
| 必須 | いいえ |
| 最大文字数 | 200文字 |
| 表示 | 現在の文字数 / 200 をリアルタイム表示 |
| バリデーションメッセージ | 「200文字以内で入力してください」 |

### 4.4 ボタン仕様

| ボタン | 動作 |
|--------|------|
| 保存 | バリデーション後にAPIへPATCHリクエスト、成功時はプロフィールページへリダイレクト |
| キャンセル | 変更を破棄してプロフィールページへ戻る（確認ダイアログは任意） |

## 5. バリデーション仕様

### クライアントサイド

- フォーム送信時に全フィールドをバリデーション
- 各フィールドのonBlur時にもバリデーション実行
- エラーはフィールド直下に赤文字で表示

### サーバーサイド（API Route / Server Action）

- クライアントと同等のバリデーションを再実施
- 認証チェック（未認証の場合は 401）
- 画像ストレージへのアップロード処理

## 6. API仕様

### PATCH /api/profile

#### リクエスト

```typescript
// multipart/form-data
{
  name: string;        // 表示名
  bio?: string;        // 自己紹介（省略可）
  avatar?: File;       // アイコン画像（省略可）
}
```

#### レスポンス（成功）

```typescript
// 200 OK
{
  id: string;
  name: string;
  bio: string | null;
  avatarUrl: string | null;
  updatedAt: string;  // ISO 8601
}
```

#### レスポンス（エラー）

```typescript
// 400 Bad Request
{
  errors: {
    field: "name" | "bio" | "avatar";
    message: string;
  }[];
}

// 401 Unauthorized
{ message: "Unauthorized" }

// 500 Internal Server Error
{ message: "Internal Server Error" }
```

## 7. 状態管理

| 状態 | 説明 |
|------|------|
| 初期値 | 現在のプロフィール情報をAPIから取得してフォームに初期表示 |
| ローディング | 保存ボタンを無効化 + スピナー表示 |
| 成功 | トースト通知 or リダイレクト |
| エラー | エラーメッセージをフィールド下またはページ上部に表示 |

## 8. アクセシビリティ要件

- 全フォームフィールドに `label` を関連付ける（`htmlFor` / `aria-label`）
- エラーメッセージは `aria-live="polite"` で通知
- ファイル入力のカスタムボタンはキーボード操作可能にする
- 保存中は `aria-busy="true"` を設定

```
components/

```

## 10. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ | モバイル（320px〜）・タブレット・デスクトップ対応 |
| パフォーマンス | 画像プレビューはクライアントサイドで即時表示（アップロードは保存時のみ） |
| セキュリティ | CSRF対策（Next.js標準）、ファイルタイプの厳密なサーバーサイド検証 |

## 11. 未検討・今後の確認事項

- 画像ストレージの選定（例: Vercel Blob / AWS S3 / Cloudinary）
- 認証ライブラリの確認（NextAuth.js / Clerk 等）
- 既存のプロフィール取得APIのエンドポイント有無
- トースト通知ライブラリの有無（例: react-hot-toast）
