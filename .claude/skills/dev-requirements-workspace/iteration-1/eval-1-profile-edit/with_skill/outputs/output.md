# ユーザープロフィール編集 画面要件定義書

## 目的

ログイン済みユーザーが自分のプロフィール情報（名前・アイコン・自己紹介）を編集・保存できるようにする。ユーザーが自分のアカウント情報を最新の状態に保てる自己管理手段を提供する。

## 対象ユーザー

- ログイン済みの一般ユーザー
- 自分のプロフィールを更新したい場面（名前変更・アイコン差し替え・自己紹介文の更新）

## 画面構成

### プロフィール編集画面（`/profile/edit`）

- 役割: プロフィール情報の表示・編集・保存
- 主要要素:
  - ページタイトル（「プロフィール編集」）
  - アイコン画像プレビュー（現在のアイコンを表示）
  - アイコン変更ボタン（ファイル選択 or ドラッグ＆ドロップ）
  - 名前入力フィールド（テキスト）
  - 自己紹介入力フィールド（テキストエリア）
  - 保存ボタン
  - キャンセルボタン（編集前の状態に戻る）
  - 保存成功トースト通知
  - バリデーションエラーメッセージ（各フィールド直下）
- 遷移先:
  - 保存成功後 → プロフィール表示画面（`/profile`）またはトップページ
  - キャンセル → 直前の画面（戻る）

### 補助 UI

| UI 要素 | 用途 |
|---------|------|
| トースト通知 | 保存完了・エラー発生をユーザーに通知 |
| ローディングスピナー | 保存処理中・画像アップロード中の待機表示 |
| 画像クロップモーダル（Nice to Have） | アイコン画像の切り抜き・調整 |

## 機能リスト

| 機能 | 優先度 | 備考 |
|------|--------|------|
| 名前の表示・編集 | Must | 現在の名前を初期表示する |
| 自己紹介の表示・編集 | Must | 現在の自己紹介を初期表示する |
| アイコン画像の表示 | Must | 現在のアイコンをプレビュー表示する |
| アイコン画像のアップロード | Must | ファイル選択から画像を変更できる |
| 保存処理（フォーム送信） | Must | 変更内容をサーバーに保存する |
| 名前のバリデーション（必須・文字数制限） | Must | 空文字不可、最大 50 文字 |
| 自己紹介のバリデーション（文字数制限） | Must | 最大 200 文字（任意入力） |
| アイコン画像のバリデーション（形式・サイズ） | Must | JPEG/PNG/WebP、最大 5MB |
| 保存成功トースト通知 | Must | 保存完了をユーザーに伝える |
| バリデーションエラーメッセージ表示 | Must | 各フィールド直下にインライン表示 |
| ローディング状態（保存中・アップロード中） | Must | ボタン無効化 + スピナー表示 |
| キャンセル動作（変更を破棄して戻る） | Should | 未保存の変更がある場合は確認ダイアログを表示 |
| アイコン画像のドラッグ＆ドロップ対応 | Should | UX 向上のため |
| アイコン画像のクロップ機能 | Nice to Have | 画像のトリミング・調整 |
| アイコン削除（デフォルト画像に戻す） | Nice to Have | アイコンをデフォルトに戻せる |

## 状態定義

| 状態 | 表示内容 |
|------|---------|
| 初期表示 | 現在のプロフィール情報をフォームに表示 |
| 編集中 | ユーザーが入力中（特別な UI 変化なし） |
| 保存処理中 | 保存ボタンを無効化しローディングスピナーを表示 |
| 保存成功 | 成功トーストを表示し、プロフィール表示画面へ遷移 |
| バリデーションエラー | 問題のあるフィールド直下にエラーメッセージを表示 |
| 保存失敗（サーバーエラー） | エラートーストを表示し、フォームは編集可能のまま |
| 画像アップロード中 | アップロード中のローディング表示 |

## バリデーション要件

| フィールド | ルール |
|------------|--------|
| 名前 | 必須、1〜50 文字 |
| 自己紹介 | 任意、0〜200 文字 |
| アイコン画像 | 任意、JPEG / PNG / WebP、5MB 以下 |

## 非機能要件

- パフォーマンス: 保存処理は 2 秒以内に完了すること。画像アップロードが長引く場合はプログレス表示を検討する。
- アクセシビリティ: 各フォームフィールドに `<label>` を紐付ける。エラーメッセージは `aria-describedby` で対応フィールドと関連付ける。キーボード操作のみで全操作が完結できること。
- レスポンシブ対応: モバイル（375px〜）・タブレット・デスクトップで適切にレイアウトが変化すること。

## 技術スタック前提

- Next.js（App Router）+ TypeScript + Tailwind CSS
- フォームバリデーション: React Hook Form + Zod を推奨
- 画像アップロード: `<input type="file">` を起点に、サーバーアクション or API Route で処理

## 確認事項（要ユーザー確認）

以下の点は要件が曖昧なため、確認が必要です。

1. **保存後の遷移先**: 保存成功後はプロフィール表示画面（`/profile`）に遷移しますか？それとも現在の画面に留まりますか？
2. **アイコンの保存先**: 画像ファイルはどこに保存しますか？（例: Supabase Storage、S3 など）既存の画像ストレージ基盤はありますか？
3. **キャンセル時の確認ダイアログ**: 未保存の変更がある状態でキャンセルした場合、確認ダイアログを表示しますか？
4. **名前の文字数制限**: 最大 50 文字で問題ありませんか？日本語を含む場合の文字数カウント方法も確認してください。
5. **アイコン削除機能**: デフォルト画像に戻す機能は今回のスコープに含めますか？
