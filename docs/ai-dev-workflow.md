# AI駆動開発ワークフローのたたき台
## 前提
- Web開発を前提とする
- ドキュメントではなく実際のコードを正とする
  - ドキュメントはあくまで補助
- AIに与える以下の情報を事前準備する
  - 技術スタック・バージョン
  - プロジェクト構造・ディレクトリ構成
  - 命名規則・コーディング規約
  - 既存コードのパターン・慣例
  - 使用ライブラリ・ツール

## コンセプト
- アウトプット駆動開発　という開発手法を新たに定義
- アウトプットをまず最初に確定させてそれを中心に機能を開発していく
  - アウトプット＝実装前に確定させるシステムの仕様・契約
  - 該当するもの
    - 静的画面
    - APIリクエスト・レスポンスのOpenAPI定義
    - ログの形式・出力条件
    - 非同期メッセージ・イベント定義（該当する場合）
- アプリのテストコード=仕様書とする
  - ドキュメントは理解を簡単にするための補助とする

## 手順
1. issue作成
   1. AIまたは人間がissueを作成する
2. 要件定義
   1. AIが既存コードベースの調査を行う
   2. AIがアウトプットを生成する
   3. 人間がアウトプットを確認する
   4. AIと人間が相談してアウトプットを確定させる
3. 実装準備
   1. AIが2-2で決めたアウトプットを壊さないためのテストを箇条書きで書く
   2. 人間が確認する
   3. AIと人間による合意が取れたらAIが3-1で決めたテストを実装する
4. 実装
   1. AIが3-1の補助となるテストを記載する
   2. AIによる実装
   3. AIによるセルフレビュー
      1. セルフレビュー結果はAIが人間の承認を得ずに修正する
      2. 2回の修正で直せなかった場合は人間に修正できなかった旨を報告する
