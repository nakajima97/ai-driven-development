# AI駆動開発ワークフロー: Skill設計リスト

このファイルは `docs/ai-dev-workflow.md` の流れに沿って、各ステップで作成すべき skill と想定する役割を整理したものです。

## 1. 現状確認 Skill
- 目的: プロジェクトの技術的ベースラインを自動収集・まとめる
- 入力: リポジトリパス、README、package.json、環境設定ファイル
- 出力: 技術スタック一覧、依存ライブラリ、ディレクトリ構造、コーディング規約候補
- 動作:
  - 対象言語・フレームワーク判定
  - 既存の設計パターン抽出
  - 未定義項目をユーザーに問い合わせ
- 成果物（必須）:
  - 現状確認レポート（技術スタック、依存関係、ディレクトリ構造、規約候補）
- 成果物（状況に応じて）:
  - 未定義項目/追加質問リスト
  - 標準化ルール（プロジェクト次第で採用）

## 2. 要件定義 Skill
- 目的: 画面要件を構造化して整理する
- 入力: 目的、ユーザーストーリー、既存UI要素、非機能要件
- 出力: 画面構成案、機能リスト、優先度リスト
- 動作:
  - "画面が必要か" 判定のサポート
  - 画面の目的/利用シナリオをまとめる
  - API定義は行わず画面要件にフォーカス
- 成果物（必須）:
  - 画面要件定義書（画面構成案、機能リスト、優先度）
- 成果物（状況に応じて）:
  - 画面要否判定と判断根拠メモ
  - ユーザーストーリー連携マップ

## 3. 静的画面の作成 Skill
- 目的: UIモック/静的ページを生成して画面要件を固める
- 入力: 要件定義結果、デザインガイドライン
- 出力: HTML/CSS/JSまたはStorybookコンポーネント
- 動作:
  - スタイルとコンポーネント構造の提案
  - レスポンシブ対応のシナリオ提示
  - ユーザーからのフィードバック収集仕組みを含める
- 成果物（必須）:
  - UIモック成果物（HTML/CSS/JS、Storybook など）
- 成果物（状況に応じて）:
  - レスポンシブ対応チェックリスト
  - フィードバック収集手順書

## 4. 実装計画の立案 Skill
- 目的: 実装に必要なAPI/DB/アーキテクチャ計画を設計
- 入力: 静的画面確認済みの画面仕様、技術スタック
- 出力: APIインターフェイス、DBスキーマ案、タスク分解
- 動作:
  - 画面要件から必要なバックエンド機能を逆算
  - 各責務の分離（フロント/バック/ストレージ）
- 成果物（必須）:
  - 実装計画ドキュメント（API定義、DBスキーマ、タスク分解）
- 成果物（状況に応じて）:
  - アーキテクチャ図（責務分離、データフロー）

## 5. テストコード作成 Skill
- 目的: 画面を壊さないためのテストを生成
- 入力: 実装計画、画面・機能シナリオ
- 出力: ユニットテスト、統合テスト、E2Eテスト例
- 動作:
  - クリティカルパスを優先したテストケース作成
  - テスト失敗条件の確認と修正案
- 成果物（必須）:
  - テストプランとテストケース一覧（ユニット/統合/E2E）
- 成果物（状況に応じて）:
  - 失敗条件/修正案ドキュメント

## 6. コード実装 Skill
- 目的: テストグリーンを目指すコード生成・適用
- 入力: 実装計画、テスト仕様、既存コードコンテキスト
- 出力: 変更差分、PRドラフト、実装コード
- 動作:
  - 逐次コミット候補を提示
  - リネーム/フォーマットを含むCI準備
- 成果物（必須）:
  - 実装差分パッチとコード変更
- 成果物（状況に応じて）:
  - PRドラフト（必要に応じて）
  - フォーマット/リンタ適合レポート（CI方針次第で省略可）

## 7. リファクタリング Skill
- 目的: 品質改善、パフォーマンス向上、ドキュメント整備
- 入力: 完了コード、カバレッジレポート、コードレビューフィードバック
- 出力: リファクタリング案、コード改善差分
- 動作:
  - 技術負債検出、冗長削減
  - モジュール分割、抽象化強化
- 成果物（必須）:
  - リファクタリング提案書
- 成果物（状況に応じて）:
  - コード改善差分パッチ
  - ドキュメント更新内容

## 8. PR作成とレビュー依頼 Skill
- 目的: リファクタリング完了後に成果を共有し、品質保証を進める
- 入力: リファクタリング後コード、差分、テスト実行結果
- 出力: PR本文ドラフト、レビュー指示、確認リスト
- 動作:
  - PR作成手順（ブランチ名、コミットメッセージ、レビューチェックリスト）を提示
  - 影響範囲およびリスク説明を整理
  - レビューコメントへの対応方針と修正案を準備
- 成果物（必須）:
  - PRリンク・レビュー依頼メッセージ
- 成果物（状況に応じて）:
  - マージ後リリース手順・リリースノート下書き

## 追加: ハンドオフ/ジェネレーション連携 Skill
- 目的: 各Skillのアウトプットを次フェーズに渡すパイプライン操作
- 動作:
  - ステップ間の状態保存/復元
  - タスク進捗ダッシュボード更新
- 成果物（必須）:
  - ステップ間状態保存データ
- 成果物（状況に応じて）:
  - 進捗ダッシュボード更新ログ
  - 引継ぎ要約レポート
